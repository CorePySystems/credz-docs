# ERC-8004 Integration

CREDZ is built on [ERC-8004](https://eips.ethereum.org/EIPS/eip-8004), a draft standard for AI agent identity and reputation on Ethereum.

---

## What is ERC-8004?

ERC-8004 defines two registries:

1. **Identity Registry** вЂ” An ERC-721-based registry where AI agents register their on-chain identity. Each agent gets a token ID linked to a registration URI (capabilities, endpoints, metadata).

2. **Reputation Registry** вЂ” A contract where scoring systems publish reputation data for registered agents.

CREDZ implements the Reputation Registry side by computing multi-dimensional scores and publishing them on-chain.

---

## How CREDZ Extends ERC-8004

### Identity

CREDZ does **not** deploy its own Identity Registry. Instead, it reads from the existing ERC-8004 Identity Registry on Base:

```
ERC8004IdentityStub: 0x1af6A661E6B09F203F4e6cb8D13EF1fFEc1a945b (Base Mainnet)
```

When an agent registers with CREDZ via `registerAgent(erc8004Id)`, it links its ERC-8004 identity token to the CREDZ scoring system.

### Reputation

CREDZ extends the basic ERC-8004 reputation model with:

- **Multi-dimensional scores** вЂ” 5 independent domain scores instead of a single reputation value
- **Confidence levels** вЂ” Each score includes a statistical confidence measure
- **EAS attestations** вЂ” Every score update is attested via the Ethereum Attestation Service
- **On-chain verifiability** вЂ” Attestation UIDs are stored on-chain for independent verification

### Step 1: Register with ERC-8004

```solidity
IERC8004IdentityRegistry identity = IERC8004IdentityRegistry(0x1af6...);

// URI points to agent metadata (capabilities, endpoints)
uint256 tokenId = identity.register("https://myagent.ai/.well-known/agent.json");
```

The registration URI should follow the ERC-8004 agent metadata format:

```json
{
  "name": "MyAgent",
  "description": "An AI trading agent on Base",
  "capabilities": ["trading", "prediction"],
  "endpoints": {
    "api": "https://myagent.ai/api",
    "websocket": "wss://myagent.ai/ws"
  }
}
```

### Step 2: Register with CREDZ

```solidity
ICREDScoreRegistry credz = ICREDScoreRegistry(0x68b0...);
credz.registerAgent(tokenId);
```

### Step 3: Automatic Scoring

Once registered, CREDZ automatically:
1. Collects data points from on-chain activity and off-chain APIs
2. Computes domain scores using Bayesian/Wilson/Brier algorithms
3. Publishes scores on-chain via EAS attestations
4. Updates the CREDScoreRegistry

## ERC-8004 Identity Registry Interface

```solidity
interface IERC8004IdentityRegistry {
    /// @notice Register a new agent identity.
    /// @param registrationURI URI pointing to agent metadata.
    /// @return tokenId The ERC-721 token ID for this agent.
    function register(string calldata registrationURI)
        external returns (uint256 tokenId);

    /// @notice Get the registration URI for an agent.
    function registrationURI(uint256 tokenId)
        external view returns (string memory);

    /// @notice Check if an address has a registered identity.
    function isRegistered(address agent)
        external view returns (bool);

    /// @notice Get the token ID for an agent address.
    function tokenOfOwner(address agent)
        external view returns (uint256);
}
```

## Chitin Protocol Compatibility

[Chitin Protocol](https://chitin.xyz) agent passports are ERC-8004 compatible. Agents with existing Chitin identity tokens can register with CREDZ directly using their Chitin token ID вЂ” no re-registration needed.

```solidity
// If your agent already has a Chitin passport
uint256 chitinTokenId = chitinRegistry.tokenOfOwner(agentAddress);
credz.registerAgent(chitinTokenId);
```

---

## Important Notes

- ERC-8004 is currently **Draft status** вЂ” the interface may change as the EIP progresses
- CREDZ keeps ERC-8004 imports isolated to minimize migration effort if the standard evolves
- The Identity Registry on Base Mainnet is currently a stub; it will be replaced by the canonical ERC-8004 registry once the standard is finalized
